## Phase 0: Research & Technical Decisions

### Research Questions

1. **Storybook Version**: Which Storybook version is compatible with Next.js 15 and React 18?
   - **Action**: Check Storybook documentation for Next.js compatibility matrix
   - **Decision Criteria**: Use Storybook 7.6+ for Next.js 15 support, or Storybook 8 if stable

2. **Next.js Integration**: What webpack/configuration is needed for Next.js features (Image, i18n, CSS)?
   - **Action**: Review `@storybook/nextjs` preset capabilities and limitations
   - **Decision Criteria**: Use official Next.js preset if it handles Image/i18n; otherwise, custom webpack config

3. **Addon Selection**: Which addons are essential vs nice-to-have?
   - **Action**: Evaluate addon bundle size, maintenance status, and feature overlap
   - **Decision Criteria**: Essential: Controls, a11y, Viewport, Docs. Optional: Interactions, Measure

4. **Theme Switching**: How to enable light/dark mode toggle in Storybook?
   - **Action**: Test `storybook-dark-mode` addon vs custom toolbar implementation
   - **Decision Criteria**: Use addon if it integrates cleanly with project's theme system

5. **Story Organization**: Component-level vs page-level stories?
   - **Action**: Define naming convention and folder structure for stories
   - **Decision Criteria**: Prefer component-level stories for reusability; add composition examples separately

## Phase 1: Design & Contracts

### Data Model

**entities to define in data-model.md**:

1. **StoryConfig**: Metadata for component stories
   - `title`: string - Story location in sidebar (e.g., "Components/Glass/GlassCard")
   - `component`: React.ComponentType - Component being documented
   - `tags`: string[] - Auto-docs, test tags
   - `parameters`: object - Story-specific settings

2. **DecoratorContext**: Context provided to stories
   - `theme`: 'light' | 'dark' - Active theme
   - `locale`: 'en' | 'th' - Active language
   - `router`: object - Mocked Next.js router

3. **ArgTypes**: Prop control definitions
   - `control`: { type: 'select' | 'boolean' | 'text' | 'number' | 'color' }
   - `options`: any[] - Available options for select controls
   - `defaultValue`: any - Default prop value

4. **GlobalTypes**: Toolbar controls
   - `theme`: Light/dark mode switcher
   - `locale`: Language switcher (EN/TH)

### Contracts

**configuration contracts in contracts/**:

1. **main.contract.md**: Storybook main configuration
   ```typescript
   interface StorybookConfig {
     stories: string[];  // '**/*.stories.@(ts|tsx|mdx)'
     addons: string[];   // @storybook/addon-essentials, addon-a11y, etc.
     framework: {
       name: '@storybook/nextjs';
       options: { nextConfigPath: string };
     };
     staticDirs: string[];
   }
   ```

2. **decorators.contract.md**: Global decorator functions
   ```typescript
   function ThemeDecorator(Story: StoryFn, context: StoryContext): JSX.Element;
   function NextIntlDecorator(Story: StoryFn, context: StoryContext): JSX.Element;
   function NextRouterDecorator(Story: StoryFn, context: StoryContext): JSX.Element;
   ```

3. **story-template.contract.md**: Standard story structure
   ```typescript
   const meta: Meta<typeof Component> = {
     title: 'Category/Subcategory/ComponentName',
     component: Component,
     tags: ['autodocs'],
     argTypes: { /* controls */ },
   };
   
   export const Default: StoryObj<typeof Component> = {
     args: { /* default props */ },
   };
   ```

### Integration Points

- **package.json**: Add `storybook` and `build-storybook` scripts
- **TailwindCSS**: Import `globals.css` in `.storybook/preview.tsx`
- **Theme System**: Connect Storybook theme toolbar to project's theme provider
- **i18n**: Provide `next-intl` context in decorators with mock messages
- **Router**: Mock `next/navigation` hooks for components that use routing
